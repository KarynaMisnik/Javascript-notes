const input = document.getElementById("price");
        const vat = document.getElementById("vat");
        const btn = document.getElementById("btn");
        const out = document.getElementById("out");
        
        btn.addEventListener("click", calculate);
        
        function calculate(){
        
            // Check if inputs are empty
        if((input.value === "") || (vat.value === "")){
        out.innerHTML = "Input can't be empty";
        return;
        }
        
        const price = parseFloat(input.value);
        const vatValue = parseFloat(vat.value);
        
        const total = price * (1 + vatValue/100);
        out.innerHTML = (total.toFixed(2));
        }

Code Analysis and Explanation

Why if((input.value === "") || (vat.value === "")) will give NaN instead of Input can't be empty when the input is empty? input and vat are references to DOM elements, they are objects and they are (objects) NEVER equal to an empty string; IMPORTANT: even if the input is empty on the page, the element itself exists, so this comparison is meaningless. How to properly check if the user left it empty: You need the property that stores what the user typed: input.value and vat.value.
